ಈ ಕಡತವು LibreOffice ತಂತ್ರಾಂಶದ ಬಗೆಗಿನ ಪ್ರಮುಖ ಮಾಹಿತಿಯನ್ನು ಹೊಂದಿದೆ. ನೀವು ಅನುಸ್ಥಾಪನೆಯನ್ನು ಆರಂಭಿಸುವ ಮೊದಲು ಈ ಮಾಹಿತಿ ಅತ್ಯಂತ ಜಾಗರೂಕತೆಯಿಂದ ಓದಿಕೊಳ್ಳಿ ಎಂದು ಸಲಹೆ ಮಾಡಲಾಗುತ್ತದೆ.

The LibreOffice community is responsible for the development of this product, and invites you to consider participating as a community member. If you are a new user, you can visit the LibreOffice site, where you will find lots of information about the LibreOffice project and the communities that exist around it. Go to https://www.libreoffice.org/.

LibreOffice ಅನ್ನು ಎಲ್ಲರೂ ಉಚಿತವಾಗಿ ಬಳಸಬಹುದೆ?
----------------------------------------------------------------------

LibreOffice ಅನ್ನು ಯಾರು ಬೇಕಿದ್ದರೂ ಉಚಿತವಾಗಿ ಬಳಸಬಹುದಾಗಿದೆ. ನೀವು  LibreOffice ನ ಈ ಪ್ರತಿಯನ್ನು ನೀವು ಇಚ್ಛಿಸುವ ಸಂಖ್ಯೆಯ ಗಣಕಯಂತ್ರಗಳಲ್ಲಿ ಅನುಸ್ಥಾಪಿಸಬಹುದಾಗಿರುತ್ತದೆ, ಹಾಗು ನಿಮ್ಮ ಯಾವುದೆ ಉದ್ಧೇಶಗಳಲ್ಲಿ ಬಳಸಬಹುದಾಗಿರುತ್ತದೆ (ವ್ಯಾಪಾರಿ, ಸರ್ಕಾರಿ, ಸಾರ್ವಜನಿಕ ನಿರ್ವಹಣೆ ಹಾಗು ಶೈಕ್ಷಣಿಕವೂ ಸೇರಿದಂತೆ). ಹೆಚ್ಚಿನ ಮಾಹಿತಿಗಾಗಿ ಈ  LibreOffice ಡೌನ್‌ಲೋಡ್‌ನಲ್ಲಿರುವ ಪರವಾನಗಿ ಪಠ್ಯವನ್ನು ನೋಡಿ.

LibreOffice ಏಕೆ ಉಚಿತವಾಗಿದೆ?
----------------------------------------------------------------------

ನೀವು LibreOffice ನ ಈ ಪ್ರತಿಯನ್ನು ಉಚಿತವಾಗಿ ಬಳಸಬಹುದಾಗಿದೆ ಏಕೆಂದರೆ, LibreOffice ಎನ್ನುವುದು ಪ್ರತ್ಯೇಕವಾಗಿ  ನೆರವಾಗುವವರು ಹಾಗು ಕಾರ್ಪೋರೇಟ್ ಪ್ರಾಯೋಜಕರು ವಿನ್ಯಸಿಸಿ, ವಿಕಸಿಸಿ, ಪರೀಕ್ಷಿಸಿ, ಅನುವಾದಿಸಿ, ದಸ್ತಾವೇಜುಗೊಳಿಸಿ, ಬೆಂಬಲಿಸಿ, ಹಾಗು ಇನ್ನೂ ಹಲವಾರು ರೀತಿಯಲ್ಲಿ ನೆರವಾಗುವ ಮೂಲಕ ಪ್ರಪಂಚದಲ್ಲಿಯೇ ಮುಂಚೂಣಿಯಲ್ಲಿರುವ ಮನೆ ಹಾಗು ಕಛೇರಿಯಲ್ಲಿ ಬಳಸಬಹುದಾದ ಮುಕ್ತ ಉತ್ಪಾದನಾ ತಂತ್ರಾಂಶವಾಗಿ ಇಂದು ಲಭ್ಯವಿದೆ.

If you appreciate their efforts, and would like to ensure that LibreOffice continues to be available far into the future, please consider contributing to the project - see https://www.documentfoundation.org/contribution/ for details. Everyone can make a contribution of some kind.

ಅನುಸ್ಥಾಪನೆಯ ಸೂಚನೆಗಳು
----------------------------------------------------------------------

LibreOffice ಪರಿಪೂರ್ಣವಾಗಿ ಕೆಲಸ ಮಾಡಲು Java Runtime Environment (JRE) ನ ಇತ್ತೀಚಿನ ಒಂದು ಆವೃತ್ತಿಯ ಅಗತ್ಯವಿರುತ್ತದೆ. JRE ಯು LibreOffice ನ ಅನುಸ್ಥಾಪನೆಯೊಂದಿಗೆ ಬರುವುದಿಲ್ಲ, ನೀವದನ್ನು ಪ್ರತ್ಯೇಕವಾಗಿ ಅನುಸ್ಥಾಪಿಸಬೇಕಾಗುತ್ತದೆ.

ಗಣಕದ ಅಗತ್ಯತೆಗಳು
----------------------------------------------------------------------

* Microsoft Windows 7 SP1, 8, 8.1 Update (S14) or 10

ಅನುಸ್ಥಾಪನೆಯ ಪ್ರಕ್ರಿಯೆಗೆ ನಿರ್ವಹಣಾ ಹಕ್ಕುಗಳ ಅಗತ್ಯವಾಗಿರುತ್ತವೆ ಎಂಬುದನ್ನು ನೆನಪಿಡಿ.

LibreOffice ಅನ್ನು ಮೈಕ್ರೋಸಾಫ್ಟ್‍ ಆಫೀಸ್‌ ಮಾದರಿಗಳಿಗಾಗಿ ಪೂರ್ವನಿಯೋಜಿತ ಅನ್ವಯವಾಗಿಸುವ ನೋಂದಣಿಯನ್ನು ಒತ್ತಾಯಿಸಲು ಅಥವ ತಡೆಹಿಡಿಯಲು ಅನುಸ್ಥಾಪಕದೊಂದಿಗೆ ಈ ಕೆಳಗಿನ ಆಜ್ಞಾಸಾಲಿನ ಬದಲಾವಣೆಯನ್ನು ಉಪಯೋಗಿಸಿ:

* REGISTER_ALL_MSO_TYPES=1 ಎನ್ನುವುದು LibreOffice ಅನ್ನು Microsoft Office ವಿನ್ಯಾಸಗಳಿಗಾಗಿ ಪೂರ್ವನಿಯೋಜಿತ ಅನ್ವಯವಾಗಿ ಒತ್ತಾಯಪೂರ್ವಕವಾಗಿ ನೋಂದಾಯಿಸುತ್ತದೆ.
* REGISTER_NO_MSO_TYPES=1 ಎನ್ನುವುದು LibreOffice ಅನ್ನು Microsoft Office ವಿನ್ಯಾಸಗಳಿಗಾಗಿ ಪೂರ್ವನಿಯೋಜಿತ ಅನ್ವಯವಾಗಿ ನೋಂದಾಯಿಸದಂತೆ ತಡೆಯುತ್ತದೆ .

ನಿಮ್ಮ ವ್ಯವಸ್ಥೆಯಲ್ಲಿರುವ ತಾತ್ಕಾಲಿಕ ಕೋಶದಲ್ಲಿ ಸಾಕಷ್ಟು ಮುಕ್ತ ಮೆಮೊರಿಯು ಲಭ್ಯವಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ, ಹಾಗು ಅದರಿಂದ ಓದುವ, ಅದಕ್ಕೆ ಬರೆಯುವ ಹಾಗು ಚಲಾಯಿಸುವ ಅಧಿಕಾರಗಳನ್ನು ನೀಡಲಾಗಿದೆ ಎಂದು ಖಾತ್ರಿ ಮಾಡಿಕೊಳ್ಳಿ. ಅನುಸ್ಥಾಪನೆಯನ್ನು ಆರಂಭಿಸುವ ಮೊದಲು ಎಲ್ಲಾ ಪ್ರೊಗ್ರಾಮ್‌ಗಳನ್ನು ಮುಚ್ಚಿ. 

ಡಿಬಿಯನ್/ಉಬುಂಟು-ಆಧರಿತವಾದ ಲಿನಕ್ಸ್ ವ್ಯವಸ್ಥೆಯಲ್ಲಿ LibreOffice ನ ಅನುಸ್ಥಾಪನೆ
----------------------------------------------------------------------

ಒಂದು ಭಾಷಾ ಪ್ಯಾಕ್ ಅನ್ನು ಹೇಗೆ ಅನುಸ್ಥಾಪಿಸಬೇಕು ಎನ್ನುವ ಸೂಚನೆಗಾಗಿ (LibreOffice ನ US English ಆವೃತ್ತಿಯನ್ನು ಅನುಸ್ಥಾಪಿಸಿದ ನಂತರ), ಈ ಕೆಳಗಿನ Installing a Language Pack ವಿಭಾಗವನ್ನು ಓದಿ.

ಡೌನ್‌ಲೋಡ್ ಮಾಡಲಾದ ಆರ್ಕೈವ್‌ನಲ್ಲಿರುವುದನ್ನು ನೀವು ಹೊರತೆಗೆದಾಗ, ಅದರಲ್ಲಿರುವುದೆಲ್ಲವನ್ನೂ ಸಹ ಒಂದು ಉಪ-ಕೋಶಕ್ಕೆ ಹೊರತೆಗೆಯಲಾಗಿರುವುದನ್ನು ನೀವು ಕಾಣಬಹುದು. ಒಂದು ಕಡತ ವ್ಯವಸ್ಥಾಪಕ ಕಿಟಕಿಯನ್ನು ತೆರೆಯಿರಿ, ಹಾಗು ಕೋಶವನ್ನು "LibreOffice_" ಎಂದು ಆರಂಭಗೊಳ್ಳುವ ಹಾಗು ನಂತರ ಆವೃತ್ತಿ ಸಂಖ್ಯೆ ಮತ್ತು ಒಂದಿಷ್ಟು ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಮಾಹಿತಿ ಇರುವುದಕ್ಕೆ ಬದಲಾಯಿಸಿ.

ಈ ಕೋಶವು "DEBS" ಎಂಬ ಒಂದು ಉಪಕೋಶವನ್ನು ಹೊಂದಿರುತ್ತದೆ. ಕೋಶವನ್ನು "DEBS" ಕೋಶಕ್ಕೆ ಬದಲಾಯಿಸಿ.

ಕೋಶದ ಮೇಲೆ ಮೌಸ್‌ನ ಬಲಗುಂಡಿಯನ್ನು ಒತ್ತಿ ಹಾಗು "ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ತೆರೆ" ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಿ. ಒಂದು ಟರ್ಮಿನಲ್ ವಿಂಡೊವು ತೆರೆದುಕೊಳ್ಳುತ್ತದೆ. ಟರ್ಮಿನಲ್‌ ವಿಂಡೊದಲ್ಲಿನ ಆಜ್ಞಾ ಸಾಲಿನಲ್ಲಿ, ಈ ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ನಮೂದಿಸಿ (ಆಜ್ಞೆಯು ಕಾರ್ಯಗತಗೊಳ್ಳುವ ಮೊದಲು ನಿರ್ವಾಹಕ ಬಳಕೆದಾರರ ಗುಪ್ತಪದವನ್ನು ನಮೂದಿಸುವಂತೆ ನಿಮ್ಮನ್ನು ಕೇಳಲಾಗುತ್ತದೆ):

ಈ ಕೆಳಗಿನ ಆದೇಶಗಳು LibreOffice ಮತ್ತು ಗಣಕತೆರೆ ಸಂಯೋಜಿಸುವ ಪ್ಯಾಕೇಜುಗಳ ಅನುಸ್ಥಾಪಿಸುತ್ತದೆ. (ಅವುಗಳನ್ನು ನೀವು ಟೈಪ್‌ ಮಾಡುವ ಬದಲು ಅದನ್ನು  ಪ್ರತಿ ಮಾಡಿ ಟರ್ಮಿನಲ್ ತೆರೆಗೆ ಅಂಟಿಸಬಹುದು):

sudo dpkg -i *.deb

ಅನುಸ್ಥಾಪನೆಯ ಪ್ರಕ್ರಿಯೆ ಈಗ ಪೂರ್ಣಗೊಂಡಿದೆ, ಹಾಗು ಎಲ್ಲಾ LibreOffice ಅನ್ವಯಗಳ ಚಿಹ್ನೆಗಳು ಈಗ ನಿಮ್ಮ ಗಣಕತೆರೆಯ ಅನ್ವಯಗಳು/ಆಫೀಸ್ ಮೆನುವಿನಲ್ಲಿ ಇರಬೇಕು.

ಫೆಡೋರ, ಓಪನ್‌ಸುಸೆ, ಮ್ಯಾಂಡ್ರೀವ ಹಾಗು RPM ಪ್ಯಾಕೇಜುಗಳನ್ನು ಬಳಸುವ ಇತರೆ ಲಿನಕ್ಸ್‍ ವ್ಯವಸ್ಥೆಗಳಲ್ಲಿ LibreOffice ಅನ್ನು ಅನುಸ್ಥಾಪಿಸುವಿಕೆ
----------------------------------------------------------------------

ಒಂದು ಭಾಷಾ ಪ್ಯಾಕ್ ಅನ್ನು ಹೇಗೆ ಅನುಸ್ಥಾಪಿಸಬೇಕು ಎನ್ನುವ ಸೂಚನೆಗಾಗಿ (LibreOffice ನ US English ಆವೃತ್ತಿಯನ್ನು ಅನುಸ್ಥಾಪಿಸಿದ ನಂತರ), ಈ ಕೆಳಗಿನ Installing a Language Pack ವಿಭಾಗವನ್ನು ಓದಿ.

ಡೌನ್‌ಲೋಡ್ ಮಾಡಲಾದ ಆರ್ಕೈವ್‌ನಲ್ಲಿರುವುದನ್ನು ನೀವು ಹೊರತೆಗೆದಾಗ, ಅದರಲ್ಲಿರುವುದೆಲ್ಲವನ್ನೂ ಸಹ ಒಂದು ಉಪ-ಕೋಶಕ್ಕೆ ಹೊರತೆಗೆಯಲಾಗಿರುವುದನ್ನು ನೀವು ಕಾಣಬಹುದು. ಒಂದು ಕಡತ ವ್ಯವಸ್ಥಾಪಕ ಕಿಟಕಿಯನ್ನು ತೆರೆಯಿರಿ, ಹಾಗು ಕೋಶವನ್ನು "LibreOffice_" ಎಂದು ಆರಂಭಗೊಳ್ಳುವ ಹಾಗು ನಂತರ ಆವೃತ್ತಿ ಸಂಖ್ಯೆ ಮತ್ತು ಒಂದಿಷ್ಟು ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಮಾಹಿತಿ ಇರುವುದಕ್ಕೆ ಬದಲಾಯಿಸಿ.

ಈ ಕೋಶವು "RPMS"" ಎಂಬ ಒಂದು ಉಪಕೋಶವನ್ನು ಹೊಂದಿರುತ್ತದೆ. ಕೋಶವನ್ನು "RPMS"" ಕೋಶಕ್ಕೆ ಬದಲಾಯಿಸಿ.

ಕೋಶದ ಮೇಲೆ ಮೌಸ್‌ನ ಬಲಗುಂಡಿಯನ್ನು ಒತ್ತಿ ಹಾಗು "ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ತೆರೆ" ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಿ. ಒಂದು ಟರ್ಮಿನಲ್ ವಿಂಡೊವು ತೆರೆದುಕೊಳ್ಳುತ್ತದೆ. ಟರ್ಮಿನಲ್‌ ವಿಂಡೊದಲ್ಲಿನ ಆಜ್ಞಾ ಸಾಲಿನಲ್ಲಿ, ಈ ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ನಮೂದಿಸಿ (ಆಜ್ಞೆಯು ಕಾರ್ಯಗತಗೊಳ್ಳುವ ಮೊದಲು ನಿರ್ವಾಹಕ ಬಳಕೆದಾರರ ಗುಪ್ತಪದವನ್ನು ನಮೂದಿಸುವಂತೆ ನಿಮ್ಮನ್ನು ಕೇಳಲಾಗುತ್ತದೆ):

For Fedora-based systems: sudo dnf install *.rpm

ಮ್ಯಾಂಡ್ರೀವಾ-ಆಧರಿತವಾದ ವ್ಯವಸ್ಥೆಗಳಿಗಾಗಿ: sudo urpmi *.rpm

ಇತರೆ RPM-ಆಧರಿತವಾದ ವ್ಯವಸ್ಥೆಗಳಿಗಾಗಿ (ಓಪನ್‌ಸುಸೆ, ಇತರೆ): rpm -Uvh *.rpm

ಅನುಸ್ಥಾಪನೆಯ ಪ್ರಕ್ರಿಯೆ ಈಗ ಪೂರ್ಣಗೊಂಡಿದೆ, ಹಾಗು ಎಲ್ಲಾ LibreOffice ಅನ್ವಯಗಳ ಚಿಹ್ನೆಗಳು ಈಗ ನಿಮ್ಮ ಗಣಕತೆರೆಯ ಅನ್ವಯಗಳು/ಆಫೀಸ್ ಮೆನುವಿನಲ್ಲಿ ಇರಬೇಕು.

ಪರ್ಯಾಯವಾಗಿ, ಒಬ್ಬ ಬಳಕೆದಾರನಾಗಿ ಅನುಸ್ಥಾಪನೆಯನ್ನು ನಿರ್ವಹಿಸಲು ಈ ಆರ್ಕೈವ್‌ನ ಮೇಲ್ಮಟ್ಟದ ಕೋಶದಲ್ಲಿನ 'install' ಸ್ಕ್ರಿಪ್ಟನ್ನು ನೀವು ಬಳಸಬಹುದು. ಸ್ಕ್ರಿಪ್ಟ್‌  ಈ ಅನುಸ್ಥಾಪನೆಗಾಗಿ ಸಾಮಾನ್ಯವಾದ LibreOffice ಪ್ರೊಫೈಲ್‌ಗಿಂತ ಭಿನ್ನವಾದ ತನ್ನದೆ ಆದ ಪ್ರೊಫೈಲ್ ಅನ್ನು ಹೊಂದಿರುವಂತೆ LibreOffice ಅನ್ನು ಸಿದ್ಧಗೊಳಿಸುತ್ತದೆ. ಇದು ಗಣಕತೆರೆ ಮೆನು ಅಂಶಗಳು ಮತ್ತು MIME ಬಗೆಯ ನೋಂದಣಿಗಳಂತಹ ಯಾವುದೆ ವ್ಯವಸ್ಥೆ ಸಂಯೋಜನಾ ಭಾಗಗಳನ್ನು ಅನುಸ್ಥಾಪಿಸುವುದಿಲ್ಲ ಎಂಬುದನ್ನು ನೆನಪಿಡಿ.

ಲಿನಕ್ಸ್ ವಿತರಣೆಗಳಿಗಾಗಿನ ಗಣಕತೆರೆಯ ಸಂಯೋಜಿಸುವಿಕೆಯ ಕುರಿತಾದ ಟಿಪ್ಪಣಿಗಳನ್ನು ಈ ಮೇಲಿನ ಅನುಸ್ಥಾಪನಾ ಸೂಚನೆಗಳಲ್ಲಿ ಸೇರ್ಪಡಿಸಲಾಗಿಲ್ಲ.
----------------------------------------------------------------------

ಈ ಅನುಸ್ಥಾಪನಾ ಸೂಚನೆಗಳಲ್ಲಿ ನಿಶ್ಚಿತವಾಗಿ ಒಳಗೊಳ್ಳಿಸದೆ ಇರುವ ಇತರೆ ಲಿನಕ್ಸ್ ವಿತರಣೆಗಳಲ್ಲಿ LibreOffice ಅನ್ನು ಅನುಸ್ಥಾಪಿಸುವುದು ಕಷ್ಟಕರವೇನಲ್ಲ. ಗಣಕತೆರೆ ಸಂಯೋಜನೆಯು ಮಾತ್ರ ಕೊಂಚ ಭಿನ್ನವಾಗಿ ಕಾಣಿಸಿಕೊಳ್ಳಬಹುದು.

The RPMS (or DEBS, respectively) directory also contains a package named libreoffice7.4-freedesktop-menus-7.4.0.1-1.noarch.rpm (or libreoffice7.4-debian-menus_7.4.0.1-1_all.deb, respectively, or similar). This is a package for all Linux distributions that support the Freedesktop.org specifications/recommendations (https://en.wikipedia.org/wiki/Freedesktop.org), and is provided for installation on other Linux distributions not covered in the aforementioned instructions.

ಭಾಷಾ ಪ್ಯಾಕೇಜನ್ನು ಅನುಸ್ಥಾಪಿಸಲಾಗುತ್ತಿದೆ
----------------------------------------------------------------------

ನೀವು ಬಯಸುವ ಭಾಷೆ ಹಾಗು ಪ್ಲಾಟ್‌ಫಾರ್ಮನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿಕೊಳ್ಳಿ. ಅವುಗಳು ಮುಖ್ಯ ಅನುಸ್ಥಾಪನಾ ಆರ್ಕೈವುಗಳ ಲಭ್ಯವಿರುವ ಸ್ಥಳದಲ್ಲಿಯೆ ದೊರೆಯುತ್ತವೆ.  Nautilus ಕಡತ ವ್ಯವಸ್ಥಾಪಕದಿಂದ, ಡೌನ್‌ಲೋಡ್ ಮಾಡಿಕೊಳ್ಳಲಾದ ಆರ್ಕೈವ್ ಅನ್ನು ಒಂದು ಕೋಶಕ್ಕೆ ಹೊರತೆಗೆಯಿರಿ (ಉದಾಹರಣೆಗೆ ನಿಮ್ಮ ಗಣಕತೆರೆ). ನೀವು ಎಲ್ಲಾ  LibreOffice ಅನ್ವಯಗಳಿಂದಲೂ ಸಹ ನಿರ್ಗಮಿಸಿದ್ದೀರಿ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ (QuickStarter ಅನ್ನು ಆರಂಭಿಸಲಾಗಿದ್ದರೆ, ಅದನ್ನೂ ಸಹ).

ನೀವು ಡೌನ್‌ಲೋಡ್ ಮಾಡಲಾದ ಭಾಷಾ ಪ್ಯಾಕ್ ಅನ್ನು ಯಾವ ಕೋಶಕ್ಕೆ ಹೊರತೆಗೆದಿರುತ್ತೀರೊ ಅದಕ್ಕೆ ಕೋಶವನ್ನು ಬದಲಾಯಿಸಿ.

ಈಗ ಕೋಶವನ್ನು ಹೊರತೆಗೆಯುವ ಪ್ರಕ್ರಿಯೆಲ್ಲಿ ರಚಿಸಲಾದ ಕೋಶಕ್ಕೆ ಬದಲಾಯಿಸಿ. ಉದಾಹರಣೆಗೆ, ಒಂದು 32-ಬಿಟ್ ಡಿಬಿಯನ್/ಉಬುಂಟು ಆಧರಿತವಾದ ವ್ಯವಸ್ಥೆಗಾಗಿನ ಫ್ರೆಂಚ್ ಭಾಷಾ ಪ್ಯಾಕ್‌ಗಾಗಿ, ಕೋಶಕ್ಕೆ LibreOffice_, ಜೊತೆಗೆ ಒಂದಿಷ್ಟು ಆವೃತ್ತಿ ಮಾಹಿತಿ, ನಂತರ Linux_x86_langpack-deb_fr ಎಂದು ಹೆಸರಿಸಬೇಕು.

ಈಗ ಕೋಶವನ್ನು ಅನುಸ್ಥಾಪಿಸಬೇಕಿರುವ ಪ್ಯಾಕೇಜನ್ನು ಹೊಂದಿರುವ ಕೋಶಕ್ಕೆ ಬದಲಾಯಿಸಿ. ಡಿಬಿಯನ್/ಉಬುಂಟು ಆಧರಿತವಾದ ವ್ಯವಸ್ಥೆಗಳಲ್ಲಿ, ಕೋಶವು DEBS ಆಗಿರುತ್ತದೆ. ಫೆಡೋರ, ಓಪನ್‌ಸುಸೆ ಅಥವ ಮ್ಯಾಂಡ್ರಿವಾ ವ್ಯವಸ್ಥೆಗಳಲ್ಲಿ, ಕೋಶವು RPMS ಆಗಿರುತ್ತದೆ.

Nautilus ಕಡತ ವ್ಯವಸ್ಥಾಪಕದಿಂದ, ಕೋಶದ ಮೇಲೆ ಮೌಸ್‌ನ ಬಲಗುಂಡಿ-ಕ್ಲಿಕ್ ಮಾಡಿ "ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ತೆರೆ" ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಿ. ಆಗ ತೆರೆದುಕೊಳ್ಳುವ ಟರ್ಮಿನಲ್ ವಿಂಡೊದಲ್ಲಿ, ಭಾಷೆಯ ಪ್ಯಾಕ್ ಅನ್ನು ಅನುಸ್ಥಾಪಿಸಲು ಆಜ್ಞೆಯನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸಿ (ಈ ಕೆಳಗಿನ ಎಲ್ಲಾ ಆಜ್ಞೆಗಳಲ್ಲಿಯೂ ಸಹ ನಿಮ್ಮ ನಿರ್ವಾಹಕ ಬಳಕೆದಾರ ಗುಪ್ತಪದವನ್ನು ನಮೂದಿಸುವಂತೆ ಕೇಳಲಾಗಬಹುದು):

ಡಿಬಿಯನ್/ಉಬುಂಟು-ಆಧರಿತವಾದ ವ್ಯವಸ್ಥೆಗಳಲ್ಲಿ: sudo dpkg -i *.deb

For Fedora-based systems: su -c 'dnf install *.rpm'

ಮ್ಯಾಂಡ್ರೀವಾ-ಆಧರಿತವಾದ ವ್ಯವಸ್ಥೆಗಳಿಗಾಗಿ: sudo urpmi *.rpm

ಇತರೆ RPM-ಬಳಸುವ ವ್ಯವಸ್ಥೆಗಳಿಗಾಗಿ (ಓಪನ್‌ಸುಸೆ, ಇತರೆ): rpm -Uvh *.rpm

ಈಗ LibreOffice ಅನ್ವಯಗಳಲ್ಲಿ ಒಂದನ್ನು ಆರಂಭಿಸಿ - ಉದಾಹರಣೆಗೆ, ರೈಟರ್. ಉಪಕರಣಗಳು ಮೆನುಗೆ ತೆರಳಿ ಮತ್ತು ಆಯ್ಕೆಗಳನ್ನು ಆರಿಸಿ. ಆಯ್ಕೆಗಳ ಸಂವಾದ ಚೌಕದಲ್ಲಿ, "ಭಾಷೆಯ ಸಿದ್ಧತೆಗಳು" ಎನ್ನುವುದರ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ ನಂತರ "ಭಾಷೆಗಳು" ಎನ್ನುವದರ ಮೇಲೆ ಕ್ಲಿಕ್ಕಿಸಿ.  Dropdown the "ಬಳಕೆದಾರ ಸಂಪರ್ಕಸಾಧನ" ಎಂಬ ಬೀಳುಪಟ್ಟಿಯಲ್ಲಿ ಹಾಗು ನೀವು ಈಗತಾನೆ ಅನುಸ್ಥಾಪಿಸಿದ ಭಾಷೆಯನ್ನು ಆರಿಸಿ. ಬೇಕಿದ್ದಲ್ಲಿ, "ಲೊಕ್ಯಾಲ್ ಸಿದ್ಧತೆಗಳು", "ಪೂರ್ವನಿಯೋಜಿತ ಚಲಾವಣೆ ನಾಣ್ಯ", ಹಾಗು "ದಸ್ತಾವೇಜುಗಳಿಗಾಗಿನ ಪೂರ್ವನಿಯೋಜಿತ ಭಾಷೆಗಳು" ಎಂಬುವಕ್ಕೆ ಇದೇ ರೀತಿ ಮಾಡಿ.

ಈ ಸಿದ್ಧತೆಗಳನ್ನು ಹೊಂದಿಸಿದ ಮೇಲೆ, ಸರಿ ಎಂಬುದರ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ. ಸಂವಾದ ಚೌಕವು ಮುಚ್ಚಿಕೊಳ್ಳುತ್ತದೆ, ಹಾಗು LibreOffice ಇಂದ ನೀವು ನಿರ್ಗಮಿಸಿದ ನಂತರ ಮಾತ್ರ ನೀವು ಮಾಡಿದ ಬದಲಾವಣೆಗಳು ಕಾರ್ಯ ರೂಪಕ್ಕೆ ಬರುತ್ತವೆ ಎನ್ನುವ ಸಂದೇಶವು ನಿಮಗೆ ಕಾಣಿಸಿಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಅದನ್ನು ಮರಳಿ ಆರಂಭಿಸಿ (QuickStarter ಆರಂಭಗೊಂಡಿದ್ದಲ್ಲಿ ಅದನ್ನೂ ಸಹ ಮುಚ್ಚಲು ಮರೆಯಬೇಡಿ).

ಮುಂದಿನ ಬಾರಿ ನೀವು LibreOffice ಅನ್ನು ಆರಂಭಿಸಿದಾಗ, ನೀವು ಅನುಸ್ಥಾಪಿಸಿದ ಭಾಷೆಯನ್ನು ಅದು ಆರಂಭಗೊಳ್ಳುತ್ತದೆ.

ವಿಂಡೋಸ್‌ ಚಾಲಕದಲ್ಲಿನ ತೊಂದರೆಯ ಕಾರಣದಿಂದಾಗಿ, ನೀವು ALPS/ಸಿನಾಪ್ಟಿಕ್‌ ಟಚ್‌ಪ್ಯಾಡ್‌ ಅನ್ನು ಬಳಸಿಕೊಂಡು ನಿಮ್ಮ ಬೆರಳುಗಳನ್ನು ಜಾರಿಸಿದಾಗ LibreOffice ದಸ್ತಾವೇಜಿನಲ್ಲಿ ಚಲಿಸಲು (ಸ್ಕ್ರಾಲ್‌) ಸಾಧ್ಯವಿರುವುದಿಲ್ಲ.

ಟಚ್‌ಪ್ಯಾಡ್‌ ಚಲನೆಯನ್ನು ಶಕ್ತಗೊಳಿಸಲು, ಈ ಕೆಳಗಿನ ಸಾಲನ್ನು "C:\Program Files\Synaptics\SynTP\SynTPEnh.ini" ಸಂರಚನಾ ಕಡತಕ್ಕೆ ಸೇರಿಸಿ, ಹಾಗು ನಿಮ್ಮ ಗಣಕವನ್ನು ಮರಳಿ ಆರಂಭಿಸಿ:

[LibreOffice]

FC = "SALFRAME"

SF = 0x10000000

SF |= 0x00004000

ಸಂರಚನಾ ಕಡತವು ಇರುವ ಸ್ಥಳವು ವಿಂಡೋಸ್‌ನ ಬೇರೆ ಬೇರೆ ಆವೃತಿಗಳಿಗೆ ಅನುಗುಣವಾಗಿ ಬದಲಾಗಬಹುದು.

If a key combination in LibreOffice does not work as described in the LibreOffice Help, check if that shortcut is already used by the operating system. To rectify such conflicts, you can change the keys assigned by your operating system. Alternatively, you can change almost any key assignment in LibreOffice. For more information on this topic, refer to the LibreOffice Help or the Help documentation of your operating system.

Problems When Sending Documents as Emails From LibreOffice
----------------------------------------------------------------------

When sending a document via 'File - Send - Document as Email' or 'Document as PDF Attachment' problems might occur (program crashes or hangs). This is due to the Windows system file "Mapi" (Messaging Application Programming Interface) which causes problems in some file versions. Unfortunately, the problem cannot be narrowed down to a certain version number. For more information visit https://www.microsoft.com to search the Microsoft Knowledge Base for "mapi dll".
